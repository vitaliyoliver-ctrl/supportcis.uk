# SupportCIS v2 — Обзор проекта

## Назначение

Внутренний портал поддержки для команды. Заменяет v1 (21 000 строк в 20 HTML-файлах без сборки, 4 разрозненных Cloudflare Worker). Самодостаточен: Docker + PostgreSQL, не привязан к Cloudflare.

| Почта | Resend | — |
| Realtime (перерывы) | Supabase Realtime | — |
| Telegram-бот | Bot API (webhook) | — |
| Отчёты | Power Automate (URL зашит во фронт) | — |

## Тип репозитория

Multi-part: `v2/app/` (фронтенд) + `v2/worker/` (бэкенд), общий Docker-образ.

> Весь исходный код находится в папке `v2/`. Корень репозитория содержит только `CNAME`, `favicon.svg` и `_bmad/`.

## Роли пользователей

| Роль | Доступ |
|---|---|
| `operator` | Базовый — график, перерывы, продажи, тикеты, отчёты |
| `supervisor` | Оператор + редактирование графика |
| `tl` | Полный доступ, включая управление ролями, продажами, Ops, аудит |
| `ops` | TL-инструменты + Ops (кроме аудита HelpDesk) |

## Аутентификация

Без паролей: OTP-код на корпоративный email (домены из `ALLOWED_DOMAINS`). Сессия — cookie `auth_token` (7 дней). Первый вход с `OWNER_EMAIL` — автоматическая роль TL.

## Хранение данных

Все изменяемые данные хранятся в одной таблице PostgreSQL `kv(key, value, expires_at)`. Сотрудники и базовые паттерны смен — статический сид в коде (`seed.ts`, `seedNk.ts`).

## Внешние зависимости

| Сервис | Назначение | Обязателен |
|---|---|---|
| Resend | OTP-коды входа, уведомления о свапах | Да (в проде) |
| Supabase | Страница «Перерывы» (Realtime) | Зашит во фронт, работает из коробки |
| Telegram Bot API | Апрув/отказ заявок на обмен смен | Нет (опционально) |
| Power Automate | Отчёты (URL потоков зашиты в коде) | Нет |
| HelpDesk API | Тикет-система (helpdesk.com) | Нет (если нет `HELPDESK_*` в `.env`) |
