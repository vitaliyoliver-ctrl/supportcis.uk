# SupportCIS v2 — Индекс документации

> Точка входа для AI-assisted разработки. Весь код в `v2/`. Дата сканирования: 2026-06-29.

## Обзор проекта

- **Тип:** Multi-part (app + worker), Docker монорепо
- **Язык:** TypeScript (фронт и бэкенд)
- **Архитектура:** React SPA + Hono Node.js API + PostgreSQL KV-хранилище
- **Деплой:** Docker + docker-compose, самодостаточен (Cloudflare не нужен)

## Быстрый справочник

| Часть | Папка | Технологии | Точка входа |
|---|---|---|---|
| Фронтенд | `v2/app/src/` | React 18, Vite 5, TanStack Query, Tailwind | `main.tsx` → `App.tsx` |
| Бэкенд | `v2/worker/src/` | Hono 4, @hono/node-server, pg | `server.ts` (Node) → `index.ts` (маршруты) |
| БД | PostgreSQL | Одна таблица `kv` | `store.ts` (адаптер) |

## Важные замечания

1. **`supabase/migrations/001_schema.sql`** содержит нормализованную реляционную схему, которую рабочий код **не использует**. Это задел на будущую миграцию от KV к реляционной БД.
2. **Сотрудники и базовые паттерны** — статический сид в коде, не в БД. Изменение = PR.
3. **HelpDesk-интеграция** — необязательна. Если `HELPDESK_*` не заданы в `.env`, страница тикетов показывает ошибку конфигурации (503).
4. **Supabase** используется только для страницы «Перерывы» (Realtime). Зашит во фронт при сборке, не в `.env` сервера.
